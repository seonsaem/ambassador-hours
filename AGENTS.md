# Code Review Protocol

Whenever the user requests a code review ("코드 리뷰해줘" or similar), you MUST perform a comprehensive analysis by sequentially or parallelly executing the following installed skills:
1. `review`: Verify standards and spec alignments.
2. `security-review`: Conduct a deep security vulnerability scan.
3. `high-end-visual-design`: Review UI aesthetics, spring animations, and micro-interactions.
4. `review-and-refactor`: Identify refactoring targets and propose fixes.
5. `to-prd`: Ensure readiness for production deployment.

Present the aggregated findings clearly, categorized by skill.
